Module sign 

Source
Expand description

Digital signature creation (ML-DSA, SLH-DSA, FN-DSA, Ed25519, hybrid). Sign command — create digital signatures over files.

Routes through the library’s unified sign_with_key() API, which handles FIPS self-test verification, resource limits, and produces a SignedData envelope with scheme metadata + timestamp + embedded public key.

The --algorithm flag is an expert override for backward compatibility. When --public-key is provided, the unified API is used regardless.
