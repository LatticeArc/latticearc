Module encrypt 

Source
Expand description

Symmetric and hybrid encryption (AES-256-GCM, ML-KEM+X25519). Encrypt command — authenticated encryption of files.

Supports three encryption modes:

  • AES-256-GCM (SP 800-38D) — symmetric authenticated encryption using a shared 32-byte key. Provides confidentiality + integrity. Each encryption uses a random 12-byte nonce (never reused).
  • Hybrid (ML-KEM-768 + X25519 + AES-256-GCM) — post-quantum hybrid encryption using a recipient’s public key. Generates an ephemeral keypair internally.
  • PQ-only (ML-KEM + AES-256-GCM) — pure post-quantum encryption without a classical (X25519) component. Required for CNSA 2.0 compliance.

Output format: The encrypted data is written as a self-contained JSON file containing the Base64-encoded ciphertext, nonce, and algorithm metadata. This JSON format ensures the encrypted file can be stored, transmitted, and decrypted without any external state.
