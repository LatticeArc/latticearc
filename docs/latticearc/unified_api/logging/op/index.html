Stable operation-name tags for log_crypto_operation_error!.

Operators filter crypto observability logs by this tag (e.g. grepping logs by operation="hybrid_decrypt"). Defining these as const — rather than bare string literals at each call site — prevents typos from silently losing a stage’s logs.

When adding a new crypto entry point, add its tag here and use the const at all log_crypto_operation_error! call sites within that function.
