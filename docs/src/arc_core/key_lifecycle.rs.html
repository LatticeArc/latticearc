1//! SP 800-57 Key Lifecycle Management
2//!
3//! This module implements formal key lifecycle management per NIST SP 800-57
4//! requirements, including state transitions, custodianship, and audit trails.
5//!
6//! All types are defined in [`arc_types::key_lifecycle`] and re-exported here.
7//! The `transition()` method returns [`arc_types::TypeError`]; use
8//! `From<TypeError> for CoreError` (in [`crate::error`]) for seamless `?` conversion.
9//!
10//! # Key States (SP 800-57 Section 3)
11//!
12//! - **Generation**: Key material is being generated
13//! - **Active**: Key is ready for use
14//! - **Rotating**: Key rotation in progress (overlap period)
15//! - **Retired**: Key scheduled for retirement
16//! - **Destroyed**: Key material zeroized
17//!
18//! # Example
19//!
20//! ```
21//! use arc_core::key_lifecycle::{KeyLifecycleRecord, KeyLifecycleState};
22//!
23//! let mut record = KeyLifecycleRecord::new(
24//!     "key-123".to_string(),
25//!     "ML-KEM-768".to_string(),
26//!     3,   // security level
27//!     365, // rotation interval (days)
28//!     30,  // overlap period (days)
29//! );
30//!
31//! // Activate the key
32//! record.transition(
33//!     KeyLifecycleState::Active,
34//!     "alice".to_string(),
35//!     "Key generation complete".to_string(),
36//!     Some("approval-123".to_string()),
37//! ).expect("Valid transition");
38//!
39//! assert!(record.is_valid_for_use());
40//! ```
