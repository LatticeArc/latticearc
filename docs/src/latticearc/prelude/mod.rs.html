1//! LatticeArc Prelude
2//!
3//! Common types, traits, and utilities used throughout LatticeArc.
4//! Provides error handling, domain constants, testing infrastructure, AND
5//! the most-frequently-used unified-API surface re-exported for ergonomic
6//! `use latticearc::prelude::*` consumption.
7//!
8//! # Example
9//!
10//! ```rust,no_run
11//! use latticearc::prelude::*;
12//!
13//! // NOTE: `Result` in this prelude is the single-generic alias
14//! // `Result<T> = std::result::Result<T, LatticeArcError>` for legacy
15//! // callers. New code working against the unified API should use
16//! // `std::result::Result<T, CoreError>` directly (aliased as
17//! // `StdResult` below to disambiguate).
18//! # use std::result::Result as StdResult;
19//! # fn example() -> StdResult<(), CoreError> {
20//! let config = CryptoConfig::new();
21//! let key = vec![0x42u8; 32];
22//! let plaintext = b"hello, world";
23//! let encrypted = encrypt(plaintext, EncryptKey::Symmetric(&key), config)?;
24//! # Ok(())
25//! # }
26//! ```
27//!
28//! # What's in this prelude
29//!
30//! - **Unified API entry points:** `encrypt`, `decrypt`, `sign_with_key`, `verify`,
31//!   `generate_signing_keypair`, `generate_hybrid_keypair`.
32//! - **Configuration / key types:** `CryptoConfig`, `EncryptKey`, `DecryptKey`,
33//!   `SecurityLevel`, `EncryptedOutput`, `SignedData`.
34//! - **Error type:** `CoreError` (returned by every API entry point).
35//! - **Compatibility re-exports:** `LatticeArcError`, `Result` (the prelude-error
36//!   `Result<T> = Result<T, LatticeArcError>` alias for legacy callers; new code
37//!   should use `Result<T, CoreError>` directly).
38//!
39//! Test infrastructure modules (`cavp_compliance`, `formal_verification`,
40//! `ci_testing_framework`, `memory_safety_testing`,
41//! `property_based_testing`, `side_channel_analysis`) are gated behind
42//! `cfg(any(test, feature = "test-utils"))`. They contain hardcoded
43//! reference vectors, KAT seeds, and validation harnesses that have no
44//! place in a release build of the cryptographic library — shipping them
45//! unconditionally was bloating release artifacts and embedding
46//! non-product code in deployed binaries. Downstream crates that need
47//! these utilities at build-time should opt in via the `test-utils`
48//! Cargo feature.
