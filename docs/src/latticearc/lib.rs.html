6//! LatticeArc - Post-Quantum Cryptography Library
7//!
8//! Post-quantum cryptography on the aws-lc-rs FIPS 140-3 backend. Hybrid
9//! ML-KEM+X25519 by default, all 4 NIST PQC standards (FIPS 203–206) — one
10//! crate, zero unsafe. For PQ TLS, use rustls 0.23.37+ with `aws-lc-rs`
11//! directly; this crate does not wrap rustls.
12//!
13//! > **IMPORTANT — FIPS terminology:** this crate distinguishes *algorithm
14//! > conformance* (implementing the NIST specs FIPS 203/204/205/206) from
15//! > *module validation* (FIPS 140-3 CMVP certification of a cryptographic
16//! > module). LatticeArc implements the algorithms for all four standards but
17//! > is **not itself CMVP-validated**. With `--features fips`, AES-GCM,
18//! > ML-KEM, HKDF, and SHA-2 are routed through the FIPS 140-3 validated
19//! > aws-lc-rs backend. ML-DSA (`fips204`), SLH-DSA (`fips205`), and FN-DSA
20//! > (`fn-dsa`) are NIST-conformant but use non-validated crate
21//! > implementations regardless of feature flags.
22//!
23//! ## Why LatticeArc?
24//!
25//! | Without LatticeArc | With LatticeArc |
26//! |--------------------|-----------------|
27//! | ~50 lines for hybrid encrypt | 3 lines |
28//! | Manage 4 key vectors manually | Single [`EncryptKey::Hybrid`] |
29//! | Research NIST parameter sets | [`UseCase`] auto-selects |
30//! | Manual secret zeroization | Automatic via `Zeroize` |
31//!
32//! ## Known limitations
33//!
34//! - **No streaming AEAD API.** All `encrypt` / `decrypt` entry points
35//!   operate on whole-message buffers. For very large payloads (multi-GiB
36//!   files, network streams), partition into chunks at the application
37//!   layer and bind chunk-index + total-count into the AEAD additional
38//!   authenticated data so the per-chunk tags compose into a single
39//!   document tag. A native streaming API is on the roadmap; not in 0.8.
40//! - **No async API.** Cryptographic functions are blocking. They run in
41//!   microseconds-to-low-milliseconds and don't interact with I/O, so
42//!   wrapping them in `tokio::task::spawn_blocking` is the recommended
43//!   integration pattern. A native async API is not planned because the
44//!   added complexity is rarely worth the wins given the latency profile.
45//!
46//! ## Algorithm Validation Status
47//!
48//! | Algorithm | Standard | Backend | FIPS Validated |
49//! |-----------|----------|---------|----------------|
50//! | ML-KEM | FIPS 203 | aws-lc-rs | Yes |
51//! | AES-256-GCM | SP 800-38D | aws-lc-rs | Yes |
52//! | HKDF-SHA256 | SP 800-56C | aws-lc-rs | Yes |
53//! | SHA-256 | FIPS 180-4 | aws-lc-rs | Yes |
54//! | ML-DSA | FIPS 204 | fips204 | No |
55//! | SLH-DSA | FIPS 205 | fips205 | No |
56//! | FN-DSA | FIPS 206 (draft / Falcon) | fn-dsa | No |
57//!
58//! ## Unified API with CryptoConfig
59//!
60//! All cryptographic operations use [`CryptoConfig`] for configuration. This builder
61//! pattern provides automatic algorithm selection based on use case or security level,
62//! with optional Zero Trust session verification.
63//!
64//! ### Basic Usage (Hybrid — Recommended)
65//!
66//! ```rust,no_run
67//! # fn main() -> Result<(), Box<dyn std::error::Error>> {
68//! use latticearc::{encrypt, decrypt, CryptoConfig, EncryptKey, DecryptKey};
69//!
70//! // Hybrid encryption: ML-KEM-768 + X25519 + HKDF + AES-256-GCM
71//! let (pk, sk) = latticearc::generate_hybrid_keypair()?;
72//! let encrypted = encrypt(b"secret", EncryptKey::Hybrid(&pk), CryptoConfig::new())?;
73//! let decrypted = decrypt(&encrypted, DecryptKey::Hybrid(&sk), CryptoConfig::new())?;
74//! # Ok(())
75//! # }
76//! ```
77//!
78//! ### PQ-Only Encryption (0.6.0+)
79//!
80//! Use [`CryptoMode::PqOnly`] for pure post-quantum encryption without a classical
81//! component. Required for CNSA 2.0 compliance.
82//!
83//! ```rust,no_run
84//! # fn main() -> Result<(), Box<dyn std::error::Error>> {
85//! use latticearc::{encrypt, decrypt, CryptoConfig, CryptoMode, EncryptKey, DecryptKey};
86//!
87//! // PQ-only: ML-KEM-768 + HKDF + AES-256-GCM (no X25519)
88//! let (pk, sk) = latticearc::generate_pq_keypair()
89//!     .map_err(|e| Box::new(e) as Box<dyn std::error::Error>)?;
90//! let config = CryptoConfig::new().crypto_mode(CryptoMode::PqOnly);
91//! let encrypted = encrypt(b"secret", EncryptKey::PqOnly(&pk), config.clone())?;
92//! let decrypted = decrypt(&encrypted, DecryptKey::PqOnly(&sk), config)?;
93//! # Ok(())
94//! # }
95//! ```
96//!
97//! For low-level access to the PQ-only ciphertext components, use
98//! [`PqOnlyCiphertext`](hybrid::pq_only::PqOnlyCiphertext) getters or `into_parts()`.
99//! To convert a hybrid scheme to its PQ-only equivalent at the same NIST level:
100//! [`EncryptionScheme::to_pq_equivalent()`].
101//!
102//! ## Digital Signatures
103//!
104//! ```rust,no_run
105//! # fn main() -> Result<(), Box<dyn std::error::Error>> {
106//! use latticearc::{generate_signing_keypair, sign_with_key, verify, CryptoConfig};
107//!
108//! let message = b"Document to sign";
109//!
110//! // Generate a persistent signing keypair (ML-DSA-65 + Ed25519 hybrid)
111//! let (pk, sk, scheme) = generate_signing_keypair(CryptoConfig::new())?;
112//!
113//! // Sign with the persistent keypair
114//! let signed = sign_with_key(message, &sk, &pk, CryptoConfig::new())?;
115//!
116//! // Verify (uses public key embedded in SignedData)
117//! let is_valid = verify(&signed, CryptoConfig::new())?;
118//! # Ok(())
119//! # }
120//! ```

122//! ## Feature Flags
123//!
124//! | Feature | Description |
125//! |---------|-------------|
126//! | `fips` | FIPS 140-3 validated backend via aws-lc-rs. Requires CMake + Go build tools. Without this feature, aws-lc-rs uses its default non-FIPS backend (C compiler only). Transitively enables `fips-self-test`. |
127//! | `fips-self-test` | Power-up KAT self-tests (ML-KEM via aws-lc-rs, AES-GCM, SHA-3, ML-DSA, SLH-DSA, FN-DSA). Pulled in transitively by `fips`; can be enabled standalone for non-FIPS builds that still want POST coverage. |
128//! | `tracing-init` | Subscriber/init helpers (`init_tracing`, `init_tracing_with_file`) for the `tracing` facade. Off by default so downstream binaries that wire their own subscriber don't fight ours. The `latticearc-cli` binary enables this. |
129//! | `zkp-serde` | Serialization support for ZKP types (enables `serde_with` for Schnorr/Sigma protocol structs). |
130//! | `secret-mlock` | OS-level memory locking for `SecretVec` (Secret Type Invariant I-10). Uses `mlock(2)` on Linux/macOS via `region`. **No-op on Windows** — `VirtualLock` is documented as best-effort and `VirtualUnlock`'s `ERROR_NOT_LOCKED` path causes a `panic = "abort"` build to abort the process; the round-17 audit fix gates this feature off on Windows targets so the cfg evaluates `false` regardless of the feature flag. Default-off because `RLIMIT_MEMLOCK` can fail at constructor time on some deployments. |
131//! | `kat-test-vectors` | Exposes `AeadCipher::new_allow_weak_key` (AES-GCM Test Cases 1/2 reproducer). Default-off so production builds cannot construct weak-key ciphers. |
132//! | `test-utils` | Exposes FIPS module-error-state recovery helpers (`clear_error_state`, `restore_operational_state`) for negative tests that need to recover global state without re-running POST. FIPS 140-3 §9.6 state-machine bypass — must NOT be enabled in production builds. Default-off. (Note: earlier rounds listed `SigmaProof::challenge_mut` here. Round-35 M3 narrowed it to `#[cfg(test)]` — strictly narrower than `test-utils` — so downstream consumers **cannot** acquire the soundness-bypass mutator by enabling this feature. The mutator is reachable only from in-tree lib unit tests.) |
133//! | `kat-replay` | Exposes legitimate KAT-replay entry points that bypass operator parameter floors but NOT cryptographic soundness checks. Currently: `pbkdf2_kat` (replays RFC 6070 / NIST CAVP fixtures with sub-OWASP iteration counts; DoS cap and PRF correctness still enforced). The CLI enables this for `kdf --allow-weak-iterations`. Distinct from `test-utils`; safe to ship to consumers that need vector replay. Default-off. |
134//! | `formal-verification` | Compilation marker: enables formal verification harness code (Kani proofs). Does not run proofs — use `cargo kani` separately. |
135//! | `kani` | Compilation marker: enables Kani bounded model checking proof harnesses. Requires `cargo kani` to execute proofs. |
136//! | `saw` | Compilation marker: enables SAW formal verification markers (inherited from aws-lc-rs). Does not run SAW proofs at build time. |
137//!
138//! ## More Examples
139//!
140//! ### Symmetric Encryption
141//!
142//! ```rust,no_run
143//! # fn main() -> Result<(), Box<dyn std::error::Error>> {
144//! use latticearc::{encrypt, decrypt, CryptoConfig, CryptoScheme, EncryptKey, DecryptKey};
145//! use latticearc::primitives::rand::random_bytes;
146//!
147//! // Generate a fresh 256-bit key from the OS CSPRNG. Never use a constant
148//! // (e.g. `[0u8; 32]`) — AEAD constructors reject all-zero keys per the
149//! // McGrew/Viega NIST AES-GCM Test Cases 1 and 2 weak-key check.
150//! let key = random_bytes(32);
151//! let encrypted = encrypt(b"secret", EncryptKey::Symmetric(&key),
152//!     CryptoConfig::new().force_scheme(CryptoScheme::Symmetric))?;
153//! let decrypted = decrypt(&encrypted, DecryptKey::Symmetric(&key), CryptoConfig::new())?;
154//! # Ok(())
155//! # }
156//! ```
157//!
158//! ### With Use Case Selection
159//!
160//! ```rust,no_run
161//! # fn main() -> Result<(), Box<dyn std::error::Error>> {
162//! use latticearc::{encrypt, CryptoConfig, UseCase, EncryptKey};
163//! use latticearc::generate_hybrid_keypair_with_level;
164//! use latticearc::primitives::kem::ml_kem::MlKemSecurityLevel;
165//!
166//! // FileStorage resolves to ML-KEM-1024 (Level 5), so the keypair MUST
167//! // be generated at the matching level. `generate_hybrid_keypair()`
168//! // defaults to ML-KEM-768 and would be rejected by
169//! // `validate_key_matches_scheme`.
170//! let (pk, _sk) = generate_hybrid_keypair_with_level(MlKemSecurityLevel::MlKem1024)?;
171//! let encrypted = encrypt(b"data", EncryptKey::Hybrid(&pk),
172//!     CryptoConfig::new().use_case(UseCase::FileStorage))?;
173//! # Ok(())
174//! # }
175//! ```
176//!
177//! ### Zero Trust Session Verification
178//!
179//! For production deployments, use [`VerifiedSession`] to enable Zero Trust
180//! verification before each operation:
181//!
182//! ```rust,no_run
183//! # fn main() -> Result<(), Box<dyn std::error::Error>> {
184//! use latticearc::{
185//!     encrypt, decrypt, CryptoConfig, VerifiedSession, generate_keypair,
186//!     EncryptKey, DecryptKey,
187//! };
188//!
189//! // Step 1: Generate a keypair (done once, typically at provisioning)
190//! let (pk, sk) = generate_keypair()?;
191//!
192//! // Step 2: Establish a verified session (performs challenge-response)
193//! let session = VerifiedSession::establish(pk.as_slice(), sk.expose_secret())?;
194//!
195//! // Step 3: Hybrid encryption with session verification
196//! let (enc_pk, enc_sk) = latticearc::generate_hybrid_keypair()?;
197//! let encrypted = encrypt(b"secret", EncryptKey::Hybrid(&enc_pk),
198//!     CryptoConfig::new().session(&session))?;
199//! let decrypted = decrypt(&encrypted, DecryptKey::Hybrid(&enc_sk),
200//!     CryptoConfig::new().session(&session))?;
201//! # Ok(())
202//! # }
203//! ```
204//!
205//! ### Hybrid Encryption (ML-KEM-768 + X25519)
206//!
207//! Use the unified API with `EncryptKey::Hybrid` / `DecryptKey::Hybrid`:
208//!
209//! ```rust,no_run
210//! # fn main() -> Result<(), Box<dyn std::error::Error>> {
211//! use latticearc::{encrypt, decrypt, CryptoConfig, EncryptKey, DecryptKey};
212//!
213//! let (pk, sk) = latticearc::generate_hybrid_keypair()?;
214//! let encrypted = encrypt(b"secret data", EncryptKey::Hybrid(&pk), CryptoConfig::new())?;
215//! let decrypted = decrypt(&encrypted, DecryptKey::Hybrid(&sk), CryptoConfig::new())?;
216//! # Ok(())
217//! # }
218//! ```
219//!
220//! ### Hybrid Signatures (ML-DSA-65 + Ed25519)
221//!
222//! ```rust,no_run
223//! # fn main() -> Result<(), Box<dyn std::error::Error>> {
224//! use latticearc::{generate_hybrid_signing_keypair, sign_hybrid, verify_hybrid_signature, SecurityMode};
225//!
226//! // Generate a hybrid signing keypair (ML-DSA-65 + Ed25519)
227//! let (pk, sk) = generate_hybrid_signing_keypair(SecurityMode::Unverified)?;
228//!
229//! // Sign (both ML-DSA and Ed25519 signatures generated)
230//! let signature = sign_hybrid(b"document", &sk, SecurityMode::Unverified)?;
231//!
232//! // Verify (both must pass for signature to be valid)
233//! let valid = verify_hybrid_signature(b"document", &signature, &pk, SecurityMode::Unverified)?;
234//! # Ok(())
235//! # }
236//! ```
237//!
238//! ### Session Lifecycle
239//!
240//! Sessions have a 30-minute default lifetime:
241//!
242//! ```rust,no_run
243//! # fn main() -> Result<(), Box<dyn std::error::Error>> {
244//! use latticearc::{encrypt, CryptoConfig, VerifiedSession, generate_keypair, CoreError, EncryptKey};
245//!
246//! let (pk, sk) = generate_keypair()?;
247//! let session = VerifiedSession::establish(pk.as_slice(), sk.expose_secret())?;
248//!
249//! // Check session properties
250//! assert!(session.is_valid());  // Not expired
251//! let _ = session.session_id(); // Unique ID for audit
252//! let _ = session.expires_at(); // Expiration time
253//!
254//! // Validate before critical operations
255//! session.verify_valid()?;  // Returns Err(SessionExpired) if expired
256//!
257//! // Refresh if expired
258//! if !session.is_valid() {
259//!     let new_session = VerifiedSession::establish(pk.as_slice(), sk.expose_secret())?;
260//! }
261//! # Ok(())
262//! # }
263//! ```
264//!
265//! ### Complete Example
266//!
267//! ```rust,no_run
268//! use latticearc::{
269//!     encrypt, decrypt, generate_signing_keypair, sign_with_key, verify,
270//!     generate_hybrid_keypair, CryptoConfig, CoreError,
271//!     EncryptKey, DecryptKey,
272//! };
273//!
274//! fn secure_workflow() -> Result<(), CoreError> {
275//!     // --- Hybrid Encryption (unified API) ---
276//!     let (enc_pk, enc_sk) = generate_hybrid_keypair()?;
277//!     let encrypted = encrypt(b"confidential", EncryptKey::Hybrid(&enc_pk),
278//!         CryptoConfig::new())?;
279//!     let decrypted = decrypt(&encrypted, DecryptKey::Hybrid(&enc_sk),
280//!         CryptoConfig::new())?;
281//!
282//!     // --- Digital Signatures ---
283//!     let (sign_pk, sign_sk, _scheme) = generate_signing_keypair(CryptoConfig::new())?;
284//!     let signed = sign_with_key(b"important document", &sign_sk, &sign_pk, CryptoConfig::new())?;
285//!     let is_valid = verify(&signed, CryptoConfig::new())?;
286//!     assert!(is_valid);
287//!
288//!     Ok(())
289//! }
290//! ```
