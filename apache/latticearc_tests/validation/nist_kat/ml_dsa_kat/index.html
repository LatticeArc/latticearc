ML-DSA Keygen Regression + Sign/Verify Roundtrip Tests

These tests use try_keygen_with_rng from the fips204 crate with a deterministic CSPRNG to generate keypairs, then verify:

  1. Keygen output matches recorded fingerprints (regression detection)
  2. Sign/verify roundtrip works correctly for each parameter set

§Why not full NIST ACVP vectors?

ML-DSA signing (FIPS 204 Algorithm 7) uses internal randomness (rnd). The fips204 crate does not expose a sign_with_rng that lets us inject test randomness, so signatures are non-deterministic. We verify keygen determinism (via try_keygen_with_rng) and sign/verify correctness (roundtrip), but cannot compare against a fixed expected signature.

§Security Levels

  • ML-DSA-44: NIST Security Level 2 (128-bit classical security)
  • ML-DSA-65: NIST Security Level 3 (192-bit classical security)
  • ML-DSA-87: NIST Security Level 5 (256-bit classical security)
