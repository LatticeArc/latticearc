Expand description

FIPS 140-3 compliant error codes for cryptographic module status indication.

These codes follow the FIPS 140-3 error reporting requirements and provide standardized error identification without revealing sensitive information.

§Error Code Ranges

  • 0x0001-0x00FF: Self-test and integrity errors (critical)
  • 0x0100-0x01FF: Algorithm errors
  • 0x0200-0x02FF: Operational errors
  • 0x0300-0x03FF: Status codes

§Example

use latticearc::primitives::fips_error::{FipsErrorCode, FipsError};

let code = FipsErrorCode::InvalidKeyLength;
assert_eq!(code.code(), 0x0100);
assert!(!code.is_critical());

// Display as FIPS-formatted string
let msg = format!("{}", code);
assert!(msg.starts_with("FIPS-0100:"));

§

SelfTestFailed = 1

Power-up self-test failed.

The cryptographic module failed its power-up self-tests as required by FIPS 140-3 Section 9.1. The module must not perform any cryptographic operations until the self-test passes.

§

IntegrityCheckFailed = 2

Software/firmware integrity check failed.

The integrity verification of the cryptographic module’s code failed. This indicates potential tampering or corruption.

§

ConditionalTestFailed = 3

Conditional self-test failed.

A conditional self-test (e.g., pairwise consistency test, continuous RNG test) failed during operation.

§

KatFailed = 4

Known Answer Test (KAT) failed.

A Known Answer Test during self-test produced incorrect results, indicating the algorithm implementation may be compromised.

§

ContinuousRngTestFailed = 5

Continuous RNG test failed.

The continuous random number generator test detected repeated output or other anomalies in the RNG.

§

InvalidKeyLength = 256

Invalid key length for the algorithm.

The provided key does not meet the length requirements for the specified algorithm.

§

InvalidNonce = 257

Invalid nonce or initialization vector.

The provided nonce/IV is invalid (wrong length, reused, or malformed).

§

DecryptionFailed = 258

Decryption failed due to authentication tag mismatch.

The ciphertext authentication failed, indicating the data was tampered with or the wrong key was used.

§

SignatureInvalid = 259

Digital signature verification failed.

The signature does not match the message and public key, indicating the message was modified or the signature is invalid.

§

InvalidParameter = 260

Invalid parameter provided to algorithm.

A parameter provided to the cryptographic operation does not meet the algorithm’s requirements.

§

UnsupportedAlgorithm = 261

Algorithm not supported or not approved.

The requested algorithm is not available in this module configuration or is not FIPS-approved.

§

KeyGenerationFailed = 262

Key generation failed.

The cryptographic key generation operation failed.

§

KEM encapsulation failed.

The Key Encapsulation Mechanism encapsulation operation failed.

§

KEM decapsulation failed.

The Key Encapsulation Mechanism decapsulation operation failed.

The digital signature generation operation failed.

Invalid public key.

Invalid secret key.

The secret/private key is malformed, has invalid length, or failed validation.

The cryptographic random number generator failed to produce output. This may indicate an entropy source failure.

The secure erasure of key material from memory could not be verified.

A required resource (memory, handles, etc.) is exhausted.

An unexpected internal error occurred in the cryptographic module.

The cryptographic module has not been initialized. Self-tests must complete before operations can be performed.

pub const fn is_critical(&self) -> bool

Checks if this is a critical error requiring module shutdown.

Critical errors indicate fundamental failures in the cryptographic module that prevent secure operation. Per FIPS 140-3, the module must cease all cryptographic operations when a critical error occurs.
