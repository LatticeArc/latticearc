pub fn secure_compare_equal_length(a: &[u8], b: &[u8]) -> bool
Expand description

Constant-time equality check for byte slices of the caller-known equal length.

§Contract

The caller is responsible for ensuring a.len() == b.len(). The function name encodes this — passing slices of different lengths returns false via a non-CT early branch on the length comparison, which is acceptable here because this function does not promise CT over variable-length inputs. A caller that compares e.g. a trimmed MAC or a variable-length signature against a known reference must not use this function — the length difference would leak through the early branch.

In debug builds the length mismatch trips a debug_assert! to surface misuse early; in release builds it returns false rather than panicking.

For the type-enforced equal-length case, prefer subtle::ConstantTimeEq::ct_eq directly on &[u8; N].
